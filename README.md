# 新闻联播文字版获取工具

获取每日央视新闻联播的文字版内容，支持查看新闻简介和详细内容，支持按月批量下载。

## 功能特点

- **智能获取**: 支持查询任意历史日期的新闻联播
- **按月下载**: 支持批量下载整月的新闻联播内容
- **新闻简介**: 快速浏览当天所有新闻标题
- **详细内容**: 查看每条新闻的完整文字内容
- **双格式输出**: 支持 TXT 和 MD 两种格式
- **网络保护**: 智能延迟机制，避免对服务器造成压力
- **并发处理**: 5线程并发获取，速度快
- **进度反馈**: 实时显示获取进度
- **智能重试**: 请求失败自动重试，带延迟策略
- **友好提示**: 获取失败时给出明确提示
- **数据来源**: 央视网官方 tv.cctv.com

## 安装

```bash
pip install requests
```

## 使用方法

### 1. 获取单日新闻简介

```bash
python get_xwlb.py today              # 查看今天的新闻简介
python get_xwlb.py yesterday          # 查看昨天的新闻简介
python get_xwlb.py 2025-09-09         # 查看指定日期的新闻简介
python get_xwlb.py 20250909           # 支持YYYYMMDD格式
```

### 2. 获取单日详细内容

```bash
python get_xwlb.py today 1            # 获取今天所有新闻详情（TXT格式）
python get_xwlb.py today 1 --md       # 获取今天所有新闻详情（MD格式）
python get_xwlb.py 2025-09-09 1       # 获取指定日期详情（TXT格式）
python get_xwlb.py 2025-09-09 1 --md  # 获取指定日期详情（MD格式）
```

### 3. 按月下载新闻

```bash
python get_xwlb.py month 2025-01      # 下载2025年1月所有新闻简介（TXT格式）
python get_xwlb.py month 2025-01 1    # 下载2025年1月所有新闻详情（TXT格式）
python get_xwlb.py month 2025-01 --md # 下载2025年1月所有新闻简介（MD格式）
python get_xwlb.py month 2025-01 1 --md # 下载2025年1月所有新闻详情（MD格式）
```

## 网络保护机制

为了避免对央视网服务器造成压力，防止IP被封，本工具实现了以下保护机制：

| 保护措施 | 参数值 | 说明 |
|---------|--------|------|
| 请求间隔 | 0.3-0.8秒 | 每个HTTP请求之间的随机延迟 |
| 每日间隔 | 1.5-3.0秒 | 按月下载时，每天之间的随机延迟 |
| 并发线程 | 5个 | 并发获取详情页，兼顾速度与稳定性 |
| 失败重试 | 2次 | 请求失败后自动重试，带延迟 |
| 重试延迟 | 2-3秒 | 重试前的等待时间 |

**建议**：
- 按月下载时，程序会自动在每天之间等待1.5-3秒
- 下载一个月大约需要 1-2 分钟（仅简介）或 5-10 分钟（含详情）
- 请勿同时运行多个下载任务

**简介文件**：
```
==================================================
XXXX年XX月XX日 新闻联播简介如下：
==================================================
第一条，这是第一条新闻标题
第二条，这是第二条新闻标题
第三条，这是第三条新闻标题
...
==================================================
（来源：XXXX年XX月XX日的新闻联播 - 央视网新闻联播主页 - https://tv.cctv.com/lm/xwlb/index.shtml）
==================================================
```

**详情文件**：
```
==================================================
第一条：这是第一条新闻标题
==================================================

　　这是第一条新闻的详细内容...

==================================================
第二条：这是第二条新闻标题
==================================================

　　这是第二条新闻的详细内容...
```

**简介文件**：
```markdown
# XXXX年XX月XX日 新闻联播

> 来源：央视网新闻联播主页 - https://tv.cctv.com/lm/xwlb/index.shtml

第一条：这是第一条新闻标题

第二条：这是第二条新闻标题

第三条：这是第三条新闻标题
```

**详情文件**：
```markdown
# XXXX年XX月XX日 新闻联播

> 来源：央视网新闻联播主页 - https://tv.cctv.com/lm/xwlb/index.shtml

## 第一条：这是第一条新闻标题

　　这是第一条新闻的详细内容...

## 第二条：这是第二条新闻标题

　　这是第二条新闻的详细内容...
```

## 技术特点

- **并发请求**: 使用 ThreadPoolExecutor 实现5线程并发获取
- **连接复用**: 使用 requests.Session 复用HTTP连接
- **智能延迟**: 随机延迟避免被识别为爬虫
- **网络保护**: 多层延迟机制保护服务器
- **智能重试**: 请求失败自动重试，带延迟
- **进度反馈**: 实时显示 `[X/Y] 已获取: 新闻标题...`
- **错误处理**: 全部失败时提示用户，部分失败时跳过并提示

## 注意事项

- 需要能够访问央视网 tv.cctv.com
- 新闻联播每天19:00播出，发布时间可能因节目时长而延迟
- **重要**: 请勿频繁请求，已内置网络保护机制
- 按月下载时请耐心等待，程序会自动控制请求频率
- 建议在非高峰时段进行批量下载
